Expand description

Draws block and creators for a given cycle. Then prune the cache pointer if max cache is exceeded. It uses BTreeMap to store the address <->rolls mapping. Although not efficient as a hashmap, it still is the best choice since the order has to be maintained.

Parameters

  • cycle: Cycle to draw
  • lookback_rolls: Roll counts at look back (cycle-3)
  • lookback_seed: RNG seed at look back (cycle-2)

Result

  • The draws can throw the errors of the function get_params and from the creation of a Xoshiro256PlusPlus.
  • An inconsistency error is thrown if nobody has rolls

Otherwise, the draws return an empty success.
